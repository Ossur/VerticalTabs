Vertical Tabs Release Notes - 0.0.6

Pinned and closed tabs are reopened at closing

When you have some tabs that are pinned and closed, then during the shutdown event of Eclipse, the tabs are reopened

It will make the tabs available after the next start of Eclipse


Changes based on issue raised in the GH 47

New menu options on Groups and Projects

You can now pin/unpin all tabs of Groups and Projects at once. You can also open all closed tabs from Group or Project

Additionally you can pin and close all tabs of the Group or Project, to let you better manage the opened tabs.
Changes based on idea raised in the GH 40

Vertical Tabs Release Notes - 0.0.5

Column for Path of file added

Column with path of the file linked to editor is now added to the view. Visibility of this column can be customized in preferences.
Changes based on idea raised in the GH 42

Columns settings

It is now possible to configure which columns and in which sequence are displayed.

The settings are available in the plugin Preferences in the Columns page.
Changes based on idea raised in the GH 41

ABAP specific features available

The update site of the plugin contains now additional feature for ABAP developers only.

It requires ADT tools to be installed.

The settings are available in the plugin Preferences in the Columns page.
Changes based on idea raised in the GH 43


After installation, two additional functionalities on the ABAP nodes will be available.

Context menu: Open in project

Ctrl+Double-Click opens list of ABAP projects in which we want open the object

Vertical Tabs Release Notes - 0.0.4

Key bindings

Key bindings for Vertical Tabsare now visible in Preferences (Global->Keys), you can customize bindings for most of the plugin commands. All of the commands have the category setup to Vertical Tabs

Focus on Vertical Tabs filter

Additionally, to the previous binidngs, focus on the Vertical Tabs filter was added - default biding Ctrl+Alt+V. It will allow you to use only keayboard if that is your style.If the view is closed at the moment of using the shortcut, it will be opened at first.
This is idea raised in the GH

Plugin preferences shortcut

The prefereces shortcut is added to view menu, for quicker access.

When used, filtered preferences dialog will be open.
